Sesi 26: Relasi & Join Database

Menghubungkan Data, Menciptakan Cerita.

1. Anatomi Tabel untuk JOIN

Agar dua tabel bisa di-JOIN, mereka harus memiliki kolom penghubung yang memiliki tipe data yang sama.

Tabel A (Kategori) PARENT TABLE

id_kategori PK nama_kategori
1Fiksi
2Sains

Tabel B (Buku) CHILD TABLE

judul id_kategori FK
Hujan1
Dunia Sophie1
Logika Hubungan: Kolom id_kategori di Tabel Buku merujuk ke id_kategori di Tabel Kategori. Angka "1" pada buku menghubungkannya ke baris "Fiksi".

2. Eksplorasi Tipe-Tipe JOIN

INNER JOIN (Pertemuan Sempurna)

Fungsi: Hanya menampilkan data jika ada pasangan yang cocok di kedua tabel. Jika sebuah buku memiliki id_kategori yang tidak ada di tabel kategori, data tersebut tidak akan muncul.

SELECT buku.judul, kategori.nama_kategori
FROM buku
INNER JOIN kategori ON buku.id_kategori = kategori.id_kategori;

LEFT JOIN (Utamakan Kiri)

Fungsi: Menampilkan SEMUA data dari tabel kiri (Buku), meskipun tidak ada pasangannya di tabel kanan (Kategori). Kolom kategori yang kosong akan diisi NULL.

SELECT buku.judul, kategori.nama_kategori
FROM buku
LEFT JOIN kategori ON buku.id_kategori = kategori.id_kategori;

3. Multi-Join: Gabungan 3 Tabel

Laporan nyata biasanya melibatkan banyak tabel. Contoh: Menampilkan Buku, Penulisnya (Wajib), dan Kategorinya (Opsional).

-- Menggabungkan 3 Tabel sekaligus
SELECT
  b.judul,
  p.nama_penulis,
  k.nama_kategori
FROM buku AS b
INNER JOIN penulis AS p ON b.id_penulis = p.id_penulis
LEFT JOIN kategori AS k ON b.id_kategori = k.id_kategori;

⚠️ Waspada: Ambiguous Column

Terjadi saat kedua tabel memiliki nama kolom yang sama (misal: keduanya punya kolom nama). Anda WAJIB menggunakan nama_tabel.nama_kolom.
