3. Aturan Penamaan (Naming Convention)

Agar database rapi dan tidak error saat dipanggil oleh PHP, ikuti aturan emas ini:

‚úÖ Gunakan snake_case
Gunakan huruf kecil dan pemisah underscore (_).
Contoh: nama_lengkap, tanggal_lahir.
‚ùå Hindari Spasi!
Spasi akan membuat query error dan sulit dipanggil.
Salah: nama siswa, tgl lahir.
‚ö†Ô∏è Hindari Keyword SQL
Jangan pakai kata perintah SQL sebagai nama tabel/kolom.
Hindari: select, table, order, date, update.
üî§ Konsisten & Jelas

4. Query Membuat Tabel (SQL)

Bahasa yang digunakan untuk "berbicara" dengan database disebut SQL (Structured Query Language).

Struktur Query: CREATE TABLE

Rumus Dasar CREATE TABLE nama_tabel ( nama_kolom1 TIPE_DATA ATURAN, nama_kolom2 TIPE_DATA ATURAN, ... );

A. Ragam Tipe Data (Wajib Tahu)

Memilih tipe data yang tepat akan menghemat memori dan mempercepat database.

üî¢ Angka Bulat
INT
Contoh: ID, Stok, Umur, Tahun.
‚öñÔ∏è Angka Desimal
FLOAT / DOUBLE
Contoh: Nilai (85.5), Berat Badan (60.2), IPK.
üíµ Uang / Presisi
DECIMAL(10,2)
Contoh: Harga (Rp 5000.00). Lebih akurat untuk uang daripada Float.
üìù Teks Pendek

Contoh: Nama, Email, Password, Username.
üìú Teks Panjang

Contoh: Alamat Lengkap, Isi Artikel, Komentar.
üìÖ Tanggal

Contoh: Tanggal Lahir (1995-08-17).
üïí Waktu Lengkap

‚úÖ Pilihan (Enum)

B. Aturan Kolom (Constraints)

Aturan ini menjaga agar data yang masuk valid dan tidak berantakan.

üîë PRIMARY KEY
Tanda pengenal utama. Data di kolom ini harus UNIK dan tidak boleh kosong. Biasanya dipakai untuk id.
ü§ñ AUTO_INCREMENT
Penomoran otomatis (1, 2, 3...). Kita tidak perlu input manual. Sangat wajib untuk id.
‚õî NOT NULL
Wajib diisi. Jika dikosongkan, database akan menolak (Error).
ü¶Ñ UNIQUE
Data tidak boleh kembar dengan data lain. Cocok untuk email atau nisn.
‚öôÔ∏è DEFAULT
Nilai bawaan jika user tidak mengisi. Contoh: Status aktif default '1', Tanggal daftar default 'NOW()'.

5. Memodifikasi Tabel (ALTER TABLE)

Seringkali di tengah jalan kita sadar ada kolom yang kurang atau salah tipe data. Kita tidak perlu menghapus tabel, cukup gunakan ALTER TABLE.

A. Menambah Kolom Baru (ADD)

Misal: Kita lupa menambahkan kolom "Email".

ALTER TABLE siswa ADD COLUMN email VARCHAR(50);

B. Mengubah Tipe Data Kolom (MODIFY)

Misal: Kolom "nama_lengkap" ternyata kurang panjang, mau diubah jadi 200 karakter.

ALTER TABLE siswa MODIFY COLUMN nama_lengkap VARCHAR(200);

C. Menghapus Kolom (DROP)

Misal: Kolom "uang_saku" tidak jadi dipakai.

ALTER TABLE siswa DROP COLUMN uang_saku;

6. Menghapus Data & Tabel

Perintah berbahaya yang harus digunakan dengan hati-hati.

Mengosongkan Isi Tabel (TRUNCATE)

Isi data hilang bersih, tapi struktur tabel masih ada (bisa diisi lagi).

TRUNCATE TABLE siswa;

Menghancurkan Tabel (DROP)

Tabel hilang sepenuhnya dari database beserta isinya.

DROP TABLE siswa;
