Sekarang, buatlah tabel peminjaman dengan struktur yang mendukung fitur "Jatuh Tempo".

Tantangan 2.1: CREATE TABLE peminjaman

  • id_pinjam (INT, PK, AI)
  • id_buku (INT)
  • nama_peminjam (VARCHAR 100)
  • tgl_pinjam (DATE)
  • tgl_kembali (DATE) -- Tanggal seharusnya buku dikembalikan
CREATE TABLE peminjaman(id_pinjam INT PRIMARY KEY AUTO INCREMENT,id_buku INT,nama_peminjam VARCHAR(100),tgl_pinjam DATE DEFAULT CURRENT TIMESTAAMP,tgl_kembali DATE);

Ilustrasi Struktur Tabel Peminjaman

üîç Fase 4: Filter & Reporting (5 Variasi)

Gunakan perintah SELECT untuk menjawab kebutuhan laporan berikut:

1. Laporan Semua Transaksi

Tampilkan seluruh kolom dari tabel peminjaman.

SELECT * FROM peminjaman

2. Filter Peminjam Spesifik

Tampilkan semua buku yang dipinjam oleh "Budi".

SELECT * FROM buku WHERE peminjam LIKE'%Budi%'

3. Filter Berdasarkan Buku

Tampilkan siapa saja yang pernah meminjam Buku dengan ID 5.

SELECT * FROM buku where buku_pinjam = 5;

4. Laporan JATUH TEMPO (Kritis!)

Tampilkan data peminjaman yang tanggal kembalinya sebelum tanggal hari ini (asumsikan hari ini 2025-11-10).

SELECT * FROM peminjaman where tgl_kembali < (2025-11-10);

5. Laporan Peminjaman Bulan Tertentu

Tampilkan semua transaksi yang terjadi pada bulan November 2025.

SELECT * FROM tranksaksi where waktu_tranksaksi BETWEEM (2025-11-01) AND (2025-11-30)
