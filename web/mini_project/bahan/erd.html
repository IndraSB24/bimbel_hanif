Skema Database Sederhana (Minimalis & Praktis)

Sesi 1: Perancangan Database

💡 Konsep Sederhana (MVP):

Kita hanya menggunakan 2 Tabel agar pemula tidak bingung.
1. Tabel Admin: Untuk sistem Login staf toko/kasir.
2. Tabel Transaksi Servis: Untuk mencatat data pelanggan, HP, keluhan, biaya, dan status pengerjaan.
Relasinya (Join): Kita mencatat id_admin di tabel transaksi untuk mengetahui siapa kasir/staf yang melayani pelanggan tersebut.

Script DDL (Create Table)

Murid bisa mengeksekusi script SQL ini di menu SQL phpMyAdmin untuk langsung membuat kedua tabel sekaligus.

-- 1. Buat tabel Admin (Sistem Login) CREATE TABLE admin ( id_admin INT PRIMARY KEY AUTO_INCREMENT, username VARCHAR(50) NOT NULL, password VARCHAR(100) NOT NULL, nama_lengkap VARCHAR(100) NOT NULL ); -- Insert data 1 admin (Kasir) default INSERT INTO admin (username, password, nama_lengkap) VALUES ('kasir01', 'rahasia123', 'Bapak Kasir Service'); -- 2. Buat tabel Transaksi Servis CREATE TABLE transaksi_servis ( id_servis INT PRIMARY KEY AUTO_INCREMENT, tgl_masuk DATE, tgl_ambil DATE, -- Tanggal HP diambil pelanggan (Bisa kosong) nama_pelanggan VARCHAR(100), no_hp VARCHAR(20), tipe_hp VARCHAR(50), keluhan TEXT, status_servis VARCHAR(20) DEFAULT 'Antrean', -- Default 'Antrean' biaya INT DEFAULT 0, -- Awal masuk biaya Rp 0 id_admin INT -- FK penanda kasir yang melayani );
